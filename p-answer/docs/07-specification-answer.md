# 07 仕様書回答版

## 1. システム概要

### システムの目的
本システムは、来店前のユーザーに対して「診断」ではなく「香りづくりの準備体験」を提供するためのフレグランスワークショップ導線である。トップページで体験価値を伝え、アンケートで好みの傾向をやわらかく可視化し、5軸グラフで微調整したうえで予約へ接続することで、来店前から香りづくりを始められる状態をつくることを目的とする。最終的な香りは店頭でスタッフと一緒に仕上げる前提を保ちつつ、スタッフへ渡す事前情報を整える役割も持つ。

### システム構成
- フロントエンド: HTML / CSS / JavaScript
- データ保持: `sessionStorage`
- 画面構成: `index.html`、`questionnaire.html`、`questionnaire_step2.html`、`fragrance-graph.html`、`reservation.html`、`reservation-complete.html`
- バックエンド連携: 現状は未実装。`reservation.html` 内に Spring Boot API 連携を想定したコメントあり

## 4. 詳細仕様

### 4-1. トップページ

- 画面ID: P-001
- 画面名: トップページ
- 目的: 来店前体験の入口として、香りワークショップの概要、流れ、安心材料、体験価値を伝え、アンケートまたは予約へ進ませる
- 主な要素:
  - ヒーローコピー「あなたに合う香りを知る」
  - CTA「香りづくりをはじめる」「そのまま予約する」
  - 3ステップ説明
  - アンケート画面・グラフ画面のプレビュー
  - 体験価値、安心材料、FAQ、最終CTA

### 4-2. STEP1アンケート

- 画面ID: P-002
- 画面名: STEP1アンケート
- 目的: 回答負荷を抑えながら共通5問でベース傾向を取得し、ユーザーに「自分向け感」を持たせる
- 主な要素:
  - 左側の質問ナビ
  - 3択カード + 中立選択肢
  - 豆知識表示
  - 回答保存: `sessionStorage` キー `fragranceStep1Answers`
- 設問:
  - Q1 惹かれる香り
  - Q2 好きな色
  - Q3 なりたい気分
  - Q4 使いたい時
  - Q5 好きな音楽

### 4-3. STEP2分岐アンケート

- 画面ID: P-003
- 画面名: STEP2アンケート
- 目的: STEP1 の回答傾向に応じた深掘り質問で、香りの印象・残り方・深み・甘さなどの方向性を補強する
- 主な要素:
  - ルート別 Q6 / Q7
  - 共通 Q8「今日の仕上がり」
  - 戻る導線、トップに戻る導線
  - 完了時に `fragrance-graph.html` へ遷移
- 分岐ルート:
  - Bloom: フローラル・甘さ寄り
  - Air: フレッシュ・透明感寄り
  - Deep: ウッディ・スパイシー寄り
  - Exploratory: 傾向が割れる場合の補助ルート

### 4-4. 香りグラフページ

- 画面ID: P-004
- 画面名: 香りグラフ
- 目的: 回答結果を視覚化し、ユーザーが気になる軸だけ自分で微調整できるようにする
- 主な要素:
  - 5軸レーダーグラフ
  - スライダー5本
  - プリセット `light` `balanced` `strong`
  - リセットボタン
  - 予約導線
- 軸:
  - floral
  - fresh
  - woody
  - spicy
  - sweet
- データ保持:
  - `sessionStorage` キー `fragranceReservationDraft`

### 4-5. 予約ページ

- 画面ID: P-005
- 画面名: 予約ページ
- 目的: 香り傾向を見ながら来店日時と来店条件を確定し、スタッフへ渡す補足情報を回収する
- 主な要素:
  - 回答傾向サマリー見出し
  - 5軸一覧表示
  - 予約枠選択モーダル
  - 来店種別、人数選択
  - スタッフ向け自由記述メモ
  - 予約確定ボタン
- 現状実装メモ:
  - 予約枠はモックデータ
  - `guestCount` と `selectedSlot` がそろうまで確定ボタンは無効
  - 確定時に `fragranceReservationConfirmation` を保存して完了画面へ遷移

### 4-6. 予約完了ページ

- 画面ID: P-006
- 画面名: 予約完了ページ
- 目的: 予約完了を明示し、予約内容、メモ、来店前案内を再確認できる状態にする
- 主な要素:
  - 完了メッセージ
  - 予約日時、種別、人数、メモの表示
  - 来店前メモ
  - 当日の流れ
  - 一定秒数後のトップページ誘導

## 5. セキュリティ・非機能

### セキュリティ観点
- 現状はフロントエンド完結で、入力内容は `sessionStorage` に保存される
- サーバー保存や認証は未実装のため、本番運用時は API 側で入力値検証、CSRF 対策、認可設計が必要
- 自由記述メモをバックエンド連携する際は XSS 対策、サニタイズ、ログ出力制御が必要

### 対応環境
- PC / スマートフォン向けレスポンシブ対応前提
- ブラウザ想定: Chrome / Edge / Safari / Firefox の最新版系
