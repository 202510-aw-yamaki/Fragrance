1 システム概要

システムの目的 必須
このページ全体が何を実現する仕組みかを簡潔に記載

本システムは、来店前のユーザーに対して「診断」ではなく「香りづくりの準備体験」を提供するフレグランスワークショップ導線です。トップページで体験価値を伝え、アンケートで好みの傾向を取得し、5軸グラフで微調整したうえで予約へ接続することで、来店前から香りづくりを始められる状態をつくります。最終的な香りは店頭でスタッフと一緒に仕上げる前提を保ちつつ、スタッフが事前に把握できる情報を整理することも目的としています。

システム構成 必須
画面構成、利用技術、データ保持方法を記載
  • フロントエンドは HTML / CSS / JavaScript で構成
  • 画面は `index.html`、`questionnaire.html`、`questionnaire_step2.html`、`fragrance-graph.html`、`reservation.html`、`reservation-complete.html` で構成
  • 回答データと予約前情報は `sessionStorage` に保持
  • 予約画面には Spring Boot API 連携を想定したコメントがあるが、現状はフロントエンド完結

4 詳細仕様

トップページ

画面の目的 必須

来店前体験の入口として、香りワークショップの概要、流れ、安心材料、体験価値を伝え、アンケートまたは予約へ進ませる。

STEP1アンケート

画面の目的 必須

回答負荷を抑えながら共通5問でベース傾向を取得し、ユーザーに自分向け感を持たせる。回答は `fragranceStep1Answers` として保存される。

STEP2アンケート

画面の目的 必須

STEP1 の回答傾向に応じて Bloom、Air、Deep、Exploratory の各ルートへ分岐し、香りの印象、残り方、深み、甘さの方向性を補強する。

香りグラフページ

画面の目的 必須

回答結果を floral、fresh、woody、spicy、sweet の5軸で可視化し、ユーザーが気になる項目だけスライダーやプリセットで微調整できるようにする。

予約ページ

画面の目的 必須

香り傾向を見ながら予約枠、来店種別、人数、スタッフ向けメモを入力させ、来店前の情報を整えた状態で予約を確定する。

予約完了ページ

画面の目的 必須

予約完了を明示し、予約日時、来店種別、人数、メモを再確認させるとともに、来店前の案内とトップページへの戻り導線を提供する。

5 セキュリティ・非機能

セキュリティ観点 必須
  • 現状はフロントエンド完結で、入力内容は `sessionStorage` に保存される
  • サーバー保存や認証は未実装のため、本番運用時は API 側で入力値検証、CSRF 対策、認可設計が必要
  • 自由記述メモをバックエンド連携する際は XSS 対策、サニタイズ、ログ出力制御が必要
対応環境 必須

PC / スマートフォン向けレスポンシブ対応を前提とし、Chrome / Edge / Safari / Firefox の最新版系ブラウザを想定する。
