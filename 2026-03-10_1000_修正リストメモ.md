# 2026-03-10 修正リストメモ

## 目的
- 主目的は診断開始率ではなく予約率の改善とする。
- 初回来店者には「1分診断から予約」、2回目以降の来店者には「診断なしで予約」の2導線を明示する。
- 画面サイズに応じて情報量を制御し、レイアウト崩れを抑えて可読性を維持する。

## 現状整理
- トップページは診断開始LPとしては成立しているが、予約率改善LPとしては導線設計が不足している。
- 再来店者向けの直予約導線が見当たらず、主要CTAがほぼ `questionnaire.html` に集中している。
- 質問ページからグラフ、予約ページまでの間で回答データが十分に連携されておらず、体験全体の説得力が弱い。
- ブラウザサイズによって情報量過多やレイアウト崩れが発生しうる構造で、表示優先度の制御も未整理。

## 優先順位

### P0: 公開前必須
- トップページのヘッダーに常時見える `予約する` 導線を追加する。
- ヒーローを二分岐化し、主CTAを `予約する`、副CTAを `初めての方はこちら（1分診断）` に変更する。
- ヒーロー直下に再来店者向けショートカットブロックを新設し、`診断せず予約する` を明示する。
- 最終CTAを予約中心に差し替え、`上へ戻る` は廃止する。
- トップページ全体の文言から「予約前に診断が必要」と読める要素を除去する。
- アンケート回答を画面遷移後も保持する仕組みを追加し、STEP1 → STEP2 → グラフ → 予約に一貫して反映させる。
- `fragrance-graph.html` の固定値ベース表示をやめ、回答内容に応じた結果生成へ変更する。
- `reservation.html` の「予約を確定する」を `alert` 仮実装から実際の予約処理または確定画面遷移へ変更する。

### P1: CV改善の中核
- 3ステップを「全員共通」ではなく「初回 / 再来店」の2ルート表示へ変更する。
- 各 `section-cta` の役割を整理し、ページ全体で `予約CTA数 >= 診断CTA数` にする。
- FAQに再来店者向け項目を追加し、「診断不要」「予約だけでよい」「当日相談可能」を明記する。
- スマホで下部固定CTAを追加し、予約導線を常時露出させる。
- CTAごとに `data-cta` 属性を付与し、位置別のクリック計測を可能にする。
- 再来店者が診断を省略してよい理由を、価値訴求テキストとして明文化する。
- 予約モーダルに初期フォーカス、フォーカストラップ、閉じた後のフォーカス復帰を実装する。
- `staff-memo` などフォーム項目のラベル不足を補い、入力意図が明確に伝わるようにする。

### P2: 体験品質と保守性
- FAQの `details/summary` 実装を見直し、現状の「全展開かつクリック不能」状態を解消する。アコーディオンを使わないなら単純な静的ブロックへ置き換える。
- 画像、カード、補足情報に表示優先度を設定し、狭いビューポートでは低優先度カードを非表示または省略表示にする。
- セクションごとに width / height 可変時の崩れを確認し、カード数・列数・余白・固定高指定を再設計する。
- 画像の `alt`、補助文言、予約ページ文言の文字化けや仮置き文言を自然な日本語へ修正する。
- 予約ページの日付表示、サマリー文言、軸ラベルなど文字化けしている箇所を修正する。
- `questionnaire.html` と `questionnaire_step2.html` の重複スクリプトとスタイルを共通化し、修正漏れリスクを下げる。
- HTML内の大きなインライン `style` / `script` を段階的に分離し、保守しやすい構成へ寄せる。

## トップページ修正指示

### 導線設計
- ヘッダーナビ末尾または右端に `予約する` を追加する。
- ヒーロー本文を「初めての方は1分診断から。2回目以降の方はそのまま予約できます。」の意味に改める。
- ヒーロー直下に `2回目以降の方へ` ブロックを追加する。
- 最終CTAは主 `予約する`、副 `初めての方は1分診断` または `診断せず予約する` にする。

### セクション別CTA
- フロー下は `初めての方は1分診断` を維持する。
- 価値訴求下は `予約する` に差し替える。
- 安心訴求下は `診断せず予約する` に差し替える。
- 最終CTAは予約中心へ寄せ、回遊目的の導線は削る。

### コピー
- `予約前に完了` のような表現は削除する。
- FAQに以下を追加する。
- `前回と同じような雰囲気で予約したいのですが、診断は必要ですか？`
- `2回目以降でも診断したほうがいいですか？`
- `予約だけ先にして、当日に相談でも大丈夫ですか？`
- 再来店者向け価値として以下を明記する。
- 前回と近い方向で相談しやすい。
- 診断を省いて時短で予約できる。
- 当日スタッフに前回のイメージを伝えやすい。
- すでに好みがある人はそのまま来店でよい。

### 計測
- CTAごとに `data-cta` を付与する。
- 想定例: `reserve-header`, `reserve-hero`, `diagnosis-hero`, `reserve-repeat-shortcut`, `reserve-final`

## 全体フロー修正指示

### 質問ページ
- 回答データを `sessionStorage` などへ保存し、ページ遷移後も復元できるようにする。
- STEP1 から STEP2 へ遷移する際に回答を引き継ぐ。
- トップへ戻る確認時、保存済み回答の扱いを整理する。

### グラフページ
- アンケート結果から5軸を算出するロジックを実装する。
- プリセットやスライダー調整後の値だけでなく、元の回答との関係がわかる状態設計にする。

### 予約ページ
- 読み込む軸データ、サマリー、日付文言の文字化けを解消する。
- 予約確定時は仮の `alert` ではなく、保存または完了画面へ遷移する。
- モーダル操作のアクセシビリティを改善する。

## レスポンシブ方針
- 各セクションに対して「必須表示」「条件付き表示」「狭幅では非表示」の優先度を定義する。
- ビューポート高が低い場合は、補助カードや装飾ビジュアルを優先的に省略する。
- ビューポート幅が狭い場合は、カードの横並び固定を避け、1カラム化と順序変更で崩れを防ぐ。
- 情報量は十分あるため、追加よりも「何を消すか」「どこで折りたたむか」を先に決める。

## 実装順の推奨
1. トップページの予約主導導線へ変更する。
2. アンケートから予約までのデータ連携を実装する。
3. 予約確定処理とモーダル操作を改善する。
4. レスポンシブ時のカード優先度制御と崩れ修正を行う。
5. 文言、FAQ、alt、ラベル、共通化などの品質改善を仕上げる。
