# WORKSHOP FLOW v2

## 香りワークショップ  
## 8問の確定版と  
## 分岐ロジック表

固定5問 → 分岐2問 → 最後の確認1問  
“自分向け感”を出しつつ、スタッフ運用は崩さない設計

- 固定 5問
- 分岐 2問
- 確認 1問

### 目的

愉快犯や冷やかし層にも  
「思ったより設計されている」と感じさせる。  
一方で、店舗側は読み取りやすい共通骨格を維持する。

## スライド 2

### 02

## なぜ「固定5問 → 分岐2問 → 確認1問」なのか

ユーザーには“見られている感”、スタッフには“読みやすさ”を両立させる。

- 固定5問で全員のベース傾向を揃えて取得する
- 分岐2問で「自分向けに聞かれている」感触をつくる
- 最後の1問で“今日の仕上がり”を微調整する
- 完全固定よりも、冷やかし層に“よく考えられている感”を与えやすい
- 完全動的よりも、スタッフ画面・分析・改善が崩れにくい

途中コピー例：  
「ここまでの回答をもとに、あと少しだけお聞きします」

## 8問の確定版：共通の固定5問

前半5問は全員共通。香り・色・気分・利用シーン・音楽からベース傾向を取る。

### Q1 今いちばん惹かれる香りは？

- 花がふわっと香る
- みずみずしく爽やか
- 木や森の落ち着き

- この中にはない

### Q2 好きな色は何ですか？

- やさしいピンク・ベージュ
- 白・水色・透明感
- ブラウン・深いグリーン
- 全部好き
- この中にはない

### Q3 どんな気分になりたいですか？

- やさしく癒やされたい
- すっきり切り替えたい
- 自分らしさを出したい
- 全部好き
- この中にはない

### Q4 どんな時に使いたいですか？

- 休日に気分を上げたい
- 仕事や外出で心地よく整えたい
- 夜や特別な時間を深く楽しみたい
- 全部好き
- この中にはない

### Q5 好きな音楽は何ですか？

- 明るいポップ
- 心にしみるストリングス
- 自然に耳に残るクラシック
- 全部好き
- この中にはない

設計意図：前半は誰でも答えやすい質問だけで構成し、後半の分岐前に離脱しないことを最優先にする。

※Q5は内部的に「ポップ=Air / ストリングス=Bloom / クラシック=Deep」として扱う。

## スライド 4

### 04

## 分岐の考え方

前半5問の傾向から、後半2問は4つのルートに分岐させる。

### A / Bloom

フローラル・スウィート寄り  
やわらかさ / 華やかさ / 甘さの出方を深掘り

### B / Air

フレッシュ寄り  
透明感 / 清潔感 / 抜け感を深掘り

### C / Deep

ウッディ・スパイシー寄り  
深み / 落ち着き / 余韻を深掘り

### D / Exploratory

傾向が割れる / 曖昧  
迷い・散り・冷やかしも拾う補助ルート

分岐により、ユーザーには「途中から自分向けに質問が変わった」感覚が生まれる。  
一方で、共通5問があるため、スタッフと分析側は全体像を見失わない。

## スライド 7

### 07

## 分岐ロジック表

固定5問の回答から、A/B/Cの傾向値と曖昧度を見て分岐先を決める。

### 基本スコア

- 各設問のメイン3択は  
  A / B / C のいずれかに +1
- 「全部好き」は  
  A / B / C に各 +0.5  
  かつ Flex +1
- 「この中にはない」は  
  加点なし  
  かつ Ambiguity +1

### 目的

“全部好き”や“この中にはない”が多い回答を、無理に決め打ちせず Exploratory に逃がせる点が重要。

### 条件 / 分岐先 / 補足

| 条件 | 分岐先 | 補足 |
|---|---|---|
| トップ1つが明確<br>(max が2以上 & 2位に1点差以上) | A / B / C のトップへ | 最も素直 |
| Flex + Ambiguity が3以上<br>かつ 1位と2位の差が1未満 | D / Exploratory | 曖昧・散り対策 |
| A と B が同点首位 | Q1 と Q4 で判定<br>爽やか / 仕事なら B<br>それ以外は A | 昼夜差を使う |
| B と C が同点首位 | Q4 で判定<br>仕事なら B / 夜なら C | シーン優先 |
| A と C が同点首位 | Q1 で判定<br>花なら A / 木なら C<br>拾えなければ D | 香り起点 |
| 全て拮抗 or 有効回答が薄い | D / Exploratory | 無理に決めない |

## スライド 8

### 08

## 店舗側に渡す情報と実装メモ

分岐の複雑さは裏側に隠し、スタッフには“読める要約”で渡す。

### スタッフ画面に出す要素

- 共通5問の回答サマリー
- 分岐先ラベル（A/B/C/D）
- 分岐2問の回答
- Q8「今日の仕上がり」
- 最終5軸グラフ（初期値 + ユーザー調整値）
- 予約ページの自由記述メモ

### 運用のポイント

分岐は“演出”でもあるが、店頭では答えの正誤として扱わない。  
「今日はこの方向から試せます」と提案の起点に使う。

### 結論

固定骨格 + 分岐演出 + 最後の確認で、体験価値と実務性を両立できる。
