## 本日起きた問題と解決法

### 1. `fragrance-graph.html` の文字化け
- 問題:
  - PowerShell の既定エンコーディングで `fragrance-graph.html` を読んだ際、日本語が文字化けして見えた
  - その状態で書き戻すと、実ファイル自体も壊れるリスクがあった
- 解決法:
  - いったん `git checkout -- fragrance-graph.html` で元状態へ戻した
  - 以後は `Get-Content -Encoding utf8` で明示的に UTF-8 読み込みを行う前提に変更
  - 日本語本文に触れず、ASCII 中心の最小差分で CSS / ヘッダーだけを修正する方針に切り替えた

### 2. `apply_patch` が文字化け行に一致しない
- 問題:
  - 文字化けして見えている行を文脈として `apply_patch` を当てると一致しないケースがあった
- 解決法:
  - まず Git に戻す
  - UTF-8 明示で内容確認
  - その上で、可能な限り ASCII のセレクタやタグ境界だけでパッチを当てる

### 3. 固定高レイアウトでセクション同士が重なる
- 問題:
  - `fragrance-graph.html` で `graph-adjustment` と `cta` が重なって見える状態が発生
  - `graph-layout` 内でも、子要素の `auto` 高さに引っ張られて親をはみ出す状況があった
- 解決法:
  - `main` を比率指定の 2 行グリッドに変更
  - `gap` を 18px に設定
  - 各カードを `min-height: 0` と `grid-template-rows: auto minmax(0, 1fr) auto` で整理
  - `helper-grid` は 3 等分に固定し、子カードも高さ制御を入れた

### 4. PC 幅が狭い時に右カラムが見切れる
- 問題:
  - `graph-layout` の最小幅が大きく、幅 768px 近辺で右端 `helper-grid` が見切れやすかった
- 解決法:
  - 3 カラムの `minmax()` を 768px 前提で再配分した
  - 現在は `minmax(220px, .84fr) minmax(300px, 1fr) minmax(160px, .58fr)` を使用

## 備考
- `fragrance-graph.html` の調整時は、今後も UTF-8 明示での確認を前提にした方が安全
- 日本語が多いファイルは、全文置換よりも「ASCII セレクタ単位での最小差分修正」を推奨
