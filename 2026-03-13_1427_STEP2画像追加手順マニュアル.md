# 2026-03-13 STEP2画像追加手順マニュアル

## 目的

- `questionnaire_step2.html` の Q6 / Q7 に画像を追加する時の進め方を固定する
- 分岐がある前提でも、迷わず整理して実装できるようにする
- 画像だけ先に差し込んで整合が崩れる事故を防ぐ

## 結論

- STEP2 は先に「分岐設計」を確定する
- その後で画像を入れる
- 画像追加は `questionnaire_step2.html` の設問データだけを触る
- CSS は原則 `css/questionnaire-common.css` の既存ルールを使う

## 参照ファイル

- `questionnaire_step2.html`
- `css/questionnaire-common.css`
- `2026-03-13_1427_STEP2分岐整理テンプレート.md`
- `2026-03-13_1242_作業中断引継ぎメモ.md`
- `2026-03-12_1704_運用マニュアル.md`

## 前提

- STEP1 は Q1 から Q5 まで画像カード化済み
- STEP2 は `renderOptions()` を STEP1 と同じ描画方式にそろえてある
- そのため、各選択肢に以下を入れれば画像カード表示できる
  - `visual`
  - `color`
  - `cardClass`

## 実装前に必ずやること

1. `branchKey` を決める
2. Q6 の各分岐内容をテンプレートに埋める
3. Q7 の各分岐内容をテンプレートに埋める
4. 画像ファイルが `img/questionnaire` にあることを確認する
5. 色を薄い単色で決める

## 推奨の進め方

### 手順1: 分岐表を先に埋める

- `2026-03-13_1427_STEP2分岐整理テンプレート.md` を使う
- 先に質問文と画像の対応を決める
- 「画像だけ決まっていて質問が未確定」の状態でコードに入らない

### 手順2: コード側の分岐構造を作る

- `questionnaire_step2.html` の `questions` を1本の固定配列で持つのではなく、必要なら分岐ごとのデータ構造へ分ける
- 例:
  - `const branchKey = ...`
  - `const questionBanks = { floral: [...], fresh: [...], woody: [...] }`
  - `const questions = questionBanks[branchKey]`

### 手順3: 各選択肢に画像を入れる

- 画像カード化する選択肢には以下を設定する

```js
{
  key: "A",
  title: "例",
  desc: "例",
  visual: "url('img/questionnaire/example.png')",
  color: "#f7f2ee",
  cardClass: "option-card-image-right"
}
```

### 手順4: 見た目確認をする

- PC 幅で画像が右寄せになっているか
- SP 幅でタイトルが画像に負けないか
- 選択時に枠や文字が崩れないか
- 画像の雰囲気と `color` が喧嘩していないか

## やってはいけないこと

- 分岐未確定のまま Q6 / Q7 の画像を直接入れ始める
- 画像ごとに個別CSSを追加する
- `background-position` や `background-size` を JS に個別追加する
- 画像表示の都合だけで HTML 構造を変える

## CSSルール

- 使用クラスは `option-card-image-right`
- 定義は `css/questionnaire-common.css`
- 効果:
  - 右寄せ
  - `background-size: auto 100%`
  - `background-repeat: no-repeat`
  - オーバーレイ無効化

## 実装判断の基準

- 同じ見せ方で済むなら共通クラスで処理する
- 微調整が必要でも、まずは `color` だけで調整する
- それでも崩れる時だけ、追加対応を検討する

## 作業単位のおすすめ

- 1. Q6 の分岐表を確定する
- 2. Q6 を実装する
- 3. 見た目を確認する
- 4. Q7 の分岐表を確定する
- 5. Q7 を実装する
- 6. 全体確認をする

## 補足

- Q8 は共通設問のまま残すと整理しやすい
- もし Q8 も分岐させるなら、Q6 / Q7 と同じテンプレート方式で別途整理する
- 作業記録を残す場合は、既存ルールどおり `YYYY-MM-DD_HHmm_内容.md` 形式で新規作成する
